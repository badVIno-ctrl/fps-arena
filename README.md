# FPS Arena

**Браузерный 3D-шутер от первого лица — боты, 1v1 и командные бои.**
Three.js на клиенте, FastAPI + WebSocket на сервере. Никаких установок — открыл ссылку и играешь.

**▶ Играть: https://fpsarena.onrender.com**

---

## Об игре

FPS Arena — динамичный 3D-шутер, который работает прямо в браузере. Без загрузок, без клиентов, без аккаунтов: открыл ссылку — и ты уже в бою. Графика на Three.js, реальный мультиплеер на WebSocket, детально смоделированное оружие (АКМ, M416), гранаты, экипировка и большая карта с укрытиями.

## Режимы игры

- **Игра с ботами** — одиночный режим. Сражайся с 60 ботами на большой карте и захвати флаг на вражеской базе. Серверное состояние не нужно — всё считается на клиенте.
- **Онлайн PvP (1v1)** — дуэль один на один. Введи ник, найди друга по нику и сражайтесь на арене в реальном времени.
- **Командный бой** — до 10 на 10. Автораспределение по командам (новый игрок попадает в меньшую), расширенная карта с укрытиями, лимит фрагов и таймер матча.

## Арсенал и экипировка

- **АКМ** и **M416** — детально смоделированное оружие с анимацией отдачи, перезарядки и прицеливания (ADS).
- **Прицелы** — переключение по `B`: открытый → коллиматор → 3x оптика.
- **Гранаты** — осколочная (`G`) и дымовая (`Y`).
- **Снаряжение** — аптечка (`3`), фонарик (`L`), глушитель (`O`), подбор предметов и пополнение патронов (`F`).
- **Стойки и движение** — спринт с расходом стамины, присед, положение лёжа, наклоны из-за угла.

## Техническое описание

FPS Arena построена по схеме «тонкий сервер — толстый клиент»: вся 3D-сцена, физика, рендеринг и боты живут в браузере, а сервер выступает быстрым ретранслятором состояний между игроками.

**Клиент**

- Three.js / WebGL — рендеринг сцены, моделей оружия, эффектов и карты.
- Один самодостаточный `game.html` со всей игровой логикой; меню вынесено в `index.html`.
- Pointer Lock для управления мышью, покадровая физика движения, стамина, стойки и баллистика.

**Сервер**

- FastAPI + Uvicorn, общение по WebSocket (`/ws`) сообщениями в формате JSON.
- Состояние в оперативной памяти и эфемерное — лобби и комнаты не пишутся в БД.
- Раздаёт статику: `/` → меню, `/game` → игра, `/sounds` → звуки выстрелов.

**Сетевая логика**

- Лобби 1v1 — поиск соперника по нику и матчмейкинг в комнату.
- Командное лобби — автобалансировка команд, старт по готовности (минимум 1 игрок в каждой команде).
- Late-join — подключение в уже идущий матч.
- Reconnect — при разрыве связи слот игрока сохраняется (по умолчанию 60 с): команда, точка спавна и счёт восстанавливаются.
- Защита счёта — идемпотентная регистрация смерти: добивающие пули и осколки не накручивают лишние фраги.
- Friendly fire выключен — урон по союзникам не засчитывается.
- Условия матча — лимит фрагов на команду, таймер и автоматическая очистка «мёртвых» комнат.

**Параметры матча** (по умолчанию, настраиваются через переменные окружения)

| Параметр | Значение | Переменная |
|---|---|---|
| Игроков в команде | 10 | TEAM_MAX |
| Лимит фрагов | 50 | MATCH_KILL_LIMIT |
| Время матча | 600 с | MATCH_TIME_LIMIT |
| Респаун | 5 с | TEAM_RESPAWN_SECS |
| Окно реконнекта | 60 с | RECONNECT_TTL |

## Технологии

- **Frontend:** Three.js, WebGL, ванильный JavaScript, HTML5
- **Backend:** Python, FastAPI, Uvicorn
- **Realtime:** WebSocket (JSON-сообщения)
- **Deploy:** Render.com (Procfile + Uvicorn)

---

## Запуск локально

```bash
git clone https://github.com/badVIno-ctrl/fps-arena.git
cd fps-arena/game
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000
```

Открой в браузере `http://localhost:8000` — и в бой.

Зависимости: `fastapi==0.115.0`, `uvicorn[standard]==0.30.6`, `websockets==13.1`
